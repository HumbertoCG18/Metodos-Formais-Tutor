| Técnicas de V&V
• Técnicas de V&V estáticas não requerem que o sistema de
software seja executado.
• Exemplo: revisões manuais de código, análise estática de padrões de código
• Técnicas de V&V dinâmicas requerem trabalhar com uma
representação executável do sistema de software.
• Exemplo: teste de software
• Técnicas de V&V formais requerem o uso de linguagens formais
de especificação e fundamentos matemáticos sólidos.
• Exemplo: prova de correção |
